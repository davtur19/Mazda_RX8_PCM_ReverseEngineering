7. KNOCK DETECTION & CONTROL
================================================================================

  Per-channel knock analysis with dual-pass state machine:

  Pass 1: MEASURE           Pass 2: RESPOND
  ┌──────────────────┐      ┌──────────────────┐
  │ pulse_window     │      │ State Machine:   │
  │ compute()        │      │ 0 = Monitoring   │
  │    ↓             │      │ 1 = Retard Active│
  │ Find max level   │      │ 2 = Recovery     │
  │ across sub-ch    │      │                  │
  │    ↓             │      │ Retard → fuel cut│
  │ Compare vs       │      │ if knock persists│
  │ threshold        │      └──────────────────┘
  │ (0x1680000)      │
  │    ↓             │
  │ KNOCK? → counter │
  └──────────────────┘

  Function: knock_detection_processing (0xF770, 0x320 bytes)
  Pipeline: knock_monitoring_pipeline_22148 → knock_detection_control_20A90

  RAM Block: 0xFFFFA4B4 - 0xFFFFA571 (per-channel, 0x58 bytes each)
  Global Counter: 0xFFFFA564 (u16, saturated)

  Tuning:
    Threshold 0x1680000: INCREASE = less sensitive, DECREASE = more sensitive
    Margin divisor 0xB40000: affects retard magnitude per event

9. REV LIMITER
================================================================================

  Three-tier protection:

  TIER 1: COLD START LIMITER (0xF11A)
    Condition: Engine cold (0xFFFFBB54 != 0)
    Action: Reduced RPM ceiling via 0x6D534 duty value
    RAM: 0xFFFFA4A6 (active), 0xFFFFA4AE (duty counter)

  TIER 2: SOFT LIMIT (0xF192)
    Threshold: Rev_Limit at 0x6D54C (stock ~9000 RPM)
    Action: Progressive fuel cut with hysteresis
    RAM: 0xFFFFA4AB (fuel cut flag), 0xFFFFA4A8 (duration counter)

  TIER 3: HARD LIMIT (0xF192)
    Threshold: Rev_Limit + offset at 0x6D550
    Action: Immediate full fuel cut
    RAM: 0xFFFFA4AC (hard limit active)

  Fuel Cut Duty: rpm_limiter_fuel_cutoff (0xC59E)
    Thresholds: 0x7A1D8 (enable), 0x7A1DC (disable), 0x7A1DE (control)
    Output: 0xFFFFA386 (duty byte)

10. DTC / DIAGNOSTICS / OBD-II
================================================================================

  DTC Enable Mechanism:
    Bitmask table at ROM 0x7E4DC, indexed by DTC number × 4
    ANDed with global RAM mask at 0xFFFFD96C
    Result: monitor enabled/disabled per DTC

  Main Gate:     obd_service_handler_6743C (routes OBD requests)
  Monitor Exec:  obd_service_handler_67494 (runs 10 monitors)

  DIAC Monitor Flags:
    0xFFFFA401  O2 Sensor (P0130/131/132) → obd_service ID 0x80
    0xFFFFA402  Catalyst Efficiency (P0420/430) → ID 0x7F
    0xFFFFA403  EGR System (P0400/401) → ID 0x81
    0xFFFFA404  EVAP System (P0440) → ID 0x82

  Catalyst DTC Disable (P0420):
    Entry at 0x7E614 (index 0x4E × 4 = offset 0x138)

11. CAN BUS COMMUNICATION
================================================================================

  Hardware: HCAN0 at 0xFFFF8000, 16 mailboxes, round-robin on 4 active
  Serial bridge: SCI1 synchronous mode, DMA channels 2 (TX) / 3 (RX)

  ⚠ CORRECTED (2026-04-05): Previously listed 0xCA/0xDC — those are internal
  mailbox/queue identifiers, NOT actual CAN bus arbitration IDs.

  Actual Mazda CAN IDs (confirmed via Ghidra cross-validation):
  TX from ECU:
    0x201  Engine RPM (bytes 0-1) + vehicle speed (4-5) + throttle (6) + status (7)
    0x203  Engine status pack
    0x215, 0x231, 0x240, 0x250, 0x251  TX data packs
    0x420  MIL/check-engine lamp status
    0x620, 0x630, 0x650  Additional TX packs
    0x41   TX pack
  RX to ECU:
    0x212  RX main handler
    0x216  RX with plausibility check + fault eval + timer
    0x430, 0x47, 0x4B0, 0x4B1, 0x4C0  RX unpack handlers

  Key Functions:
    0xD6A0  hcan_init_and_status_check (HCAN0 initialization)
    0xB770  serial_can_hardware_init (SCI/UART configuration)
    0xA57C  can_periodic_task_handler (10 Hz message scheduler)
    0xBB6C  can_serial_communication_handler (DMA-based, 582 bytes)
    0xA608  hcan0_mailbox_rotate_enqueue (round-robin mailbox selection)

  CAN Task Counter: 0xFFFFA188 (10 Hz increment)
  Init Status Flag: 0xFFFFA3FB

15. FILES FOR OPEN-SOURCE TUNING SOFTWARE
================================================================================

  To build an open-source alternative to VersaTuner/MazdaEdit, you need:

  1. TABLE DEFINITIONS — Use RX8_Calibration_Map.txt
     Maps all 1214 calibration entries with addresses and sizes.
     Combined with rx8_defs.xml from RomRaider for axis scaling.

  2. READ/WRITE PROTOCOL — UDS implementation documented in pseudocode:
     SID 0x34 (Request Download) at 0x1A70
     SID 0x36 (Transfer Data) at 0x1B8C
     SID 0x37 (Transfer Exit) at 0x1CB8
     SID 0x27 (Security Access) at 0x17D8 — SEED-KEY AUTH REQUIRED
     Flash writing covers ENTIRE ROM (not just calibration area)

  3. LIVE DATA — CAN message definitions:
     ⚠ CORRECTED: 0xCA/0xDC were internal queue IDs, not CAN arb IDs.
     Actual: 0x201 (RPM+speed+throttle), 0x203 (status), 0x420 (MIL), etc.
     OBD-II Mode $01 PIDs via standard CAN protocol

  4. SAFETY CONSTRAINTS from this RE work:
     - Never modify code section (0x0000-0x6CF00), only calibration
     - Rev limit max safe for stock 13B-MSP: ~9500 RPM
     - Ignition safe mode forces 2 deg advance (absolute minimum)
     - Lambda disable (0xFFFFA38C=1) causes open-loop (rich/lean risk)
     - DTC disable via bitmask zeroing at 0x7E4DC+offset is reversible
     - OMP tables MUST NOT be reduced (apex seal lubrication critical)
     - Cold rev limiter protects apex seals during warmup
