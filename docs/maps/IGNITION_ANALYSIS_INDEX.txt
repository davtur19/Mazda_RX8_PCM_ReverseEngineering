QUICK REFERENCE - KEY FUNCTION ADDRESSES
==========================================

PRIMARY IGNITION CONTROL
------------------------
0xE7F8   ignition_timing_calc_E7F8          Main timing calculator (0x364 bytes)
         Inputs:  RPM index, Load scalar, Knock flag, System modes
         Outputs: Leading/trailing advance, Dwell time
         Called:  Every combustion event
         Calls:   table_2d_interpolate, FPU ops, knock correction path

KNOCK DETECTION
---------------
0xF770   knock_detection_processing         Knock sensor analyzer (0x320 bytes)
         Inputs:  Knock sensor raw signal per rotor
         Outputs: Knock counter, Margin, Retard flags
         Called:  Periodically by main sensor loop
         Calls:   pulse_window_compute, threshold compare

COIL DRIVER INTERFACE
---------------------
0x884C   ign_output_calc_main               Hardware timer programming (0x244 bytes)
         Inputs:  Timing advance values from 0xFFFFA458/0xFFFFA448
         Outputs: Hardware compare register values
         Called:  Real-time spark scheduler (~every 0.25ms)
         Calls:   write_to_memory_with_fpu_sync

TIMING LIMITERS & BOUNDARIES
-----------------------------
0xE38C   ignition_advance_limiter           Safety valve (0xB4 bytes)
         Purpose: Prevents advance overflow, enforces hard limits
         Gates:   Safe mode, Acceleration enrichment checks

0x11A9C  calc_base_ignition_timing_11A9C    Master dispatcher (0xD4 bytes)
         Purpose: Orchestrates 30+ control subsystems
         Calls:   Main fuel control, knock enable, pressure sensing, etc.

0x11B70  calc_knock_retard_filter           Knock retard task dispatcher
         Purpose: Applies knock margin decay over time
         Frequency: Once per control loop iteration

0xF6E8   ignition_table_unpack_all_rotors  ROM unpacker (0x60 bytes)
         Purpose: Extracts per-rotor timing data from compact ROM storage
         Called:  Boot initialization, on-demand by test routines

FPU HELPER FUNCTIONS (called extensively)

0x20C4   fpu_sub_float                      FR0 = FR5 - FR4
0x20DC   fpu_div_float                      FR0 = FR5 / FR4
0x2120   fpu_int_to_float                   Float conversion
0x23F4   fpu_sqrt_float                     Square root (conservative cap)

TABLE LOOKUP FUNCTIONS
----------------------
0x9668   table_2d_interpolate               2D RPM × Load table lookup
0x2624   axis_search_float_array            Binary search for axis value
0x2490   axis_lookup_float_to_index         Convert float to table index

SATURATION FUNCTIONS

SYSTEM ARCHITECTURE OVERVIEW
============================

The RX-8 S1 ignition system follows a dual-spark Wankel architecture:

┌──────────────────────────────────────────────┐
│ 2 ROTORS (A & B) × 2 PLUGS EACH = 4 SPARK    │
│         CHANNELS PER 2-ROTOR CYCLE           │
└──────────────────────────────────────────────┘
          │
          ├─→ ROTOR A: Leading Channel (BTDC, aggressive)
          │   └─→ Address: 0xFFFFA458 (timing advance)
          │       Coil driver: ign_output_calc_main
          │
          └─→ ROTOR A: Trailing Channel (ATDC, cleanup)
              └─→ Address: 0xFFFFA448 (timing advance)
                  Offset: Leading + 0x6D0C4 (50-60 degrees ATDC)

CONTROL FLOW:
  Combustion event (every 180° rotor rotation)
       ↓
  Crank sensor triggers ISR
       ↓
  ignition_timing_calc_E7F8 (main timing orchestration)
       ├─→ knock_detection_processing (analyze knock sensor)
       │   └─→ Compare max energy vs 0x1680000 threshold
       │       └─→ Increment counter if exceeded
       │
       ├─→ Load RPM × Load base table (2D interpolation)
       │
       ├─→ Apply 7 conditional adjustments:
       │   1. Knock correction (gate: 0xFFFFB5AA)
       │   2. EGR penalty (gate: 0xFFFFCCE1)
       │   3. VVT adjustment (gate: 0xFFFFB340)
       │   4. Fuel mixture correction (gate: 0xFFFFA7C8)
       │   5. Boost correction (gate: 0xFFFFC600, always 0 on S1)
       │   6. Knock safety margin (gate: 0xFFFFBB38)
       │   7. Cold start boost (gate: startup flag)
       │
       ├─→ Store outputs:
       │   - Leading advance → 0xFFFFA458
       │   - Trailing advance → 0xFFFFA448
       │   - Dwell time → 0xFFFFA45C
       │
       └─→ Return to main loop

  ign_output_calc_main (convert degrees to hardware timing)
       ├─→ Read current crank angle from timer
       │
       ├─→ Calculate fire angle = current + advance
       │
       ├─→ Enforce dwell constraint (min 1.5ms charging)
       │
       ├─→ Clamp to hardware bounds (0-32767)
       │
       └─→ Write to coil compare register (atomic)

  Hardware timer interrupt (when timer == compare value)
       ├─→ Primary coil current OFF (magnetic field collapse)
       │
       ├─→ Spark plug ignition (30kV secondary voltage)
       │
       ├─→ Combustion begins
       │
       └─→ Knock sensor records detonation waves (if any)


TUNING IMPACT HIERARCHY (from high to low)
-------------------------------------------
1. Base timing table (0x6D096)          - BIGGEST impact (±40 degrees range)
2. Knock threshold (0x1680000)          - Affects knock sensitivity (freq)
3. Knock correction (0x6D0B0)           - Retard magnitude on knock (-2 to -8)
4. Cold start boost (0x6D0C6)           - Warm-up timing (+2 to +8, 10 sec only)
5. EGR penalty (0x6D0B6)                - Emission control effect (-2 to -5)
6. "VVT" adjustment (0x6D0BC)           - ⚠ NOT valve timing (Wankel misnomer)
7. Fuel mixture correction (0x6D0E2)    - O2 sensor feedback (±2 degrees)
8. Knock safety margin (0x6D0E8)        - Conservative floor (1-3 degrees)

COMMON ANALYSIS TASKS
=====================

TASK 1: "Monitor real-time timing values"
  Commands:
  - Read 0xFFFFA458 (leading channel advance, 16-bit signed) ÷ 100 = degrees BTDC
  - Read 0xFFFFA448 (trailing channel advance) ÷ 100 = degrees ATDC
  - Read 0xFFFFB5AA (knock flag: 0=none, 1=active)
  - Read 0xFFFFA496 (knock counter: 0-65535)

  Interpretation:
    If counter increasing → knock events detected
    If 0xFFFFB5AA = 1 and advance decreasing → knock retard applied

TASK 2: "Change knock sensitivity"
  Current threshold: 0x1680000 (23.6 million)
  Decrease (more sensitive): 0x1200000 (18 million) - retards sooner
  Increase (less sensitive): 0x2000000 (32 million) - allows more timing

  WARNING: Changes directly affect engine safety. Test under controlled conditions.

TASK 3: "Modify base timing curve"
  Edit 0x6D096 (leading timing table)
  Type: 2D array (IEEE 754 floats)
  Axes: RPM (10 bins, 500-8000 RPM), Load (8 bins, 0-100% MAP)

  Example: To add 2 degrees at 3000 RPM, 75% load:
    Find intersection in table, add 2.0 to float value
    Note: Also check 0x6D0A4 upper boundary (don't exceed)

TASK 4: "Trace knock detection"
  Breakpoint at 0xF770 (knock_detection_processing entry)
  For each channel:
    - Call pulse_window_compute() to extract sensor peak energy
    - Compare max_energy vs 0x1680000 threshold
    - If exceeded: increment counter at 0xFFFFA564
    - Store margin at per-rotor structure + 0x04

TASK 5: "Debug timing calculation"
  Breakpoint at 0xE7F8 (ignition_timing_calc_E7F8)
  Single-step through:
  - 0xE906: table_2d_interpolate call (base timing)
  - 0xE91A: FPU store (load-scaled result)
  - 0xE97C-0xEA00: Conditional adjustment blocks
  - 0xEA1A: Store leading advance (0xFFFFA458)
  - 0xEAE0+: Trailing synthesis (0xFFFFA448)
  - 0xEB00+: Knock counter safeguard

TASK 6: "Identify safe mode activation"
  Check:
    - 0xFFFFA4A1 = 1 (software safe mode flag)
    - 0xFFFFA4A1 = 2 (diagnostic mode flag)
    - 0xFFFFFBFA4 = 0 (test pin pulled low)

  Effect: Timing forced to 2.0 degrees BTDC, all tables skipped

TASK 7: "Verify ROM integrity"
  Checksums for key tables:
  - 0x6D096 (leading table): All values positive, increasing with RPM
  - 0x1680000 (threshold): Fixed constant
  - Boundary tables: Upper > lower (sanity check)

ABBREVIATIONS & TERMINOLOGY
=============================

BTDC     Before Top Dead Center (advance timing, ignites before TDC)
ATDC     After Top Dead Center (retard timing, ignites after TDC)
TDC      Top Dead Center (rotor face at maximum compression, 0 degrees reference)
MAP      Manifold Absolute Pressure (intake load measurement)
RPM      Revolutions Per Minute (engine speed)
WOT      Wide Open Throttle (full acceleration, high load)
EGR      Exhaust Gas Recirculation (emissions control)
VVT      ⚠ MISNOMER: "Variable Valve Timing" — Wankel has NO valves.
         Likely intake geometry or emission actuator control. Name from IDA.

O2       Oxygen sensor (rear exhaust catalyst monitor)
FPU      Floating-Point Unit (SH7055 math coprocessor)
ISR      Interrupt Service Routine (event-driven function)

RAM      Random-Access Memory (runtime state variables)
