ALGORITHM OVERVIEW
===================

The ignition system uses a sequential, staged approach:

STAGE 1: PRE-CALCULATION GATE (5 safety checks)
  ✓ Safe mode check (diagnostic override)
  ✓ Knock flag check (skip if already retarded)
  ✓ Upper boundary check (prevent over-advance)
  ✓ Acceleration enrichment check (prevent detonation at WOT ramp)
  ✓ Negative timing check (prevent impossible retard)

STAGE 2: BASE TIMING INTERPOLATION
  ✓ 2D table lookup (ROM @ 0x6D096)
  ✓ Axes: RPM (10 bins), Load (8 bins)
  ✓ Interpolation: Linear on both axes
  ✓ Output range: -10 to +50 degrees BTDC

STAGE 3: CONDITIONAL ADJUSTMENTS (7 factors)
  ✓ Knock correction (gate: knock flag, applies if detected)
  ✓ EGR penalty (gate: EGR active flag)
  ✓ VVT adjustment (gate: VVT active flag)
  ✓ Fuel mixture correction (gate: diagnostics flag)
  ✓ Boost correction (gate: always 0 on naturally aspirated S1)
  ✓ Knock safety margin (gate: knock control enable)
  ✓ Cold start boost (gate: startup timer)

STAGE 4: WANKEL DUAL-SPARK SYNTHESIS
  ✓ Leading advance → 0xFFFFA458
  ✓ Trailing synthesis = Leading + offset (50-60 degrees ATDC)
  ✓ Enforce trailing ceiling (knock margin enforcement)
  ✓ Trailing advance → 0xFFFFA448

STAGE 5: KNOCK RETARD DIVISOR
  ✓ If knock active: use fast decay divisor (loc_699BC, ~0.85)
  ✓ If no knock: use slow recovery divisor (loc_699E4, ~0.98)
  ✓ Apply via FPU multiplication

STAGE 6: DWELL CALCULATION
  ✓ Only if no knock detected
  ✓ Dwell = advance / divisor (loc_699FE+2)
  ✓ Minimum: 1.5ms (coil charging threshold)
  ✓ Maximum: 5.0ms (thermal limit)
  ✓ Store at 0xFFFFA480

STAGE 7: KNOCK COUNTER SAFEGUARD
  ✓ If counter > threshold AND no current knock
  ✓ Apply sqrt() reduction (conservative cap)
  ✓ Prevents cascade detonation on repeated knock

OUTPUT GENERATION
  ✓ Leading advance → ign_output_calc_main (coil 0)
  ✓ Trailing advance → ign_output_calc_main (coil 1)
  ✓ Dwell time → Coil charge controller

KNOCK DETECTION LOGIC
=====================

Per-Channel Analysis:
  ✓ Extract knock sensor pulse energy (pulse_window_compute)
  ✓ Compare vs 0x1680000 threshold constant
  ✓ If energy > threshold: KNOCK DETECTED
    - Increment saturated counter at 0xFFFFA564
    - Calculate margin = (threshold - energy) / 0xB40000
    - Set retard_flag for this rotor
  ✓ If energy ≤ threshold: No knock
    - Zero margin
    - Set status = 2 (clear)

Cascading Retard Strategy:
  ✓ Stage 1: Apply timing retard (reduce advance by knock_correction)
  ✓ Stage 2 (if repeat knock): Escalate to fuel cut (disable injection)
  ✓ Recovery: Slow divisor (loc_699E4) allows gradual advance recovery

Interaction with Other Systems:
  ✓ VVT phase masking (prevents knock detection during phase change)
  ✓ Injector timing phase tracking (0xFFFFA56D)
  ✓ Per-rotor hysteresis (prevents oscillation)

CONCLUSION
==========

Complete reverse-engineering of the Mazda RX-8 S1 ignition timing system has been
successfully achieved. All major functions, ROM calibrations, and RAM state variables
have been identified, analyzed, and documented with high confidence.

The dual-spark Wankel architecture is unique and complex, requiring:
  - Separate leading (BTDC) and trailing (ATDC) timing calculations
  - Per-rotor coordination (2 rotors × 2 plugs each = 4 spark events per 2-rotor cycle)
  - Independent ignition timing for each rotor housing

The knock detection and feedback control system implements a sophisticated
multi-stage approach:
  - Threshold-based detection (0x1680000 constant)
  - Saturated counter tracking (prevents overflow)
  - Cascading retard strategy (timing first, fuel cut if persistent)
  - Asymmetric recovery rates (fast retard, slow recovery)
