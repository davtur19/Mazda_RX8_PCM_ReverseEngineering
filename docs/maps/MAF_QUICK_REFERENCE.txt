================================================================================
WANKEL-SPECIFIC CONSIDERATIONS
================================================================================

Problem #1: Two Combustion Events Per Revolution
  → MAF sees doubled frequency vs. conventional piston 4-stroke
  → Solution: Tables 0x69F72-0x69FEA calibrated for Wankel pulse pattern
  → Impact: Using reciprocating-engine MAF tables = severe rich/lean shift

Problem #2: Rotor Temperature Asymmetry
  → Leading rotor ~200°C, trailing rotor ~150°C (cruise condition)
  → Solution: Separate AFR targets via rotor2 offset @ 0x6996E
  → Impact: Single AFR target = rotor2 inefficiency, emissions

Problem #3: Intake Manifold Geometry
  → Asymmetric porting = unequal charge distribution
  → Solution: Manifold pressure compensation @ 0x6ACF8, 0x6AD0C
  → Impact: Ignoring = poor idle quality, surging

Critical Flag:
  0xFFFFB5A4 = Rotor isolation (0=dual-rotor operation, 1=single rotor for diagnostics)

================================================================================
COLD-START ENRICHMENT PATH
================================================================================

Triggered When:  @(0xFFFFA6B8) == 1  (Cold-start flag)

Enrichment Sequence:
  1. Load @(0x69FAE) → Multiply MAF by cold-start factor 1
  2. Load @(0x69FC2) → Add cold-start curve
  3. Load @(0x69FD6) → Add enhanced enrichment
  4. Load @(0x69FEA) → Final post-processing

Expected Behavior:
  Cold-start (< 20°C):  Stoich × 1.5-2.0 (very rich for ignition)
  Warm-up (20-60°C):    Stoich × 1.2-1.5 (moderate enrichment)
  Warm (> 60°C):        Stoich × 1.0-1.1 (nominal to slightly rich)

Diagnostic:
  If cold-start is stuck:  Check @(0xFFFFA6B8) = 1
  If cold-start too rich:  Adjust 0x69FAE, 0x69FC2, 0x69FD6 down
  If cold-start too lean:  Adjust above tables up

TYPICAL TUNING WORKFLOW
================================================================================

STEP 1: Verify baseline (dyno fuel consumption @ WOT)
  → Measure actual AFR with wideband O2
  → Compare to 0xFFFFB5B8 reading (stoich target)
  → If mismatch > 10%, recalibrate 0xFFFFB5B8

STEP 2: Optimize closed-loop response (cruise steady-state)
  → Verify 0xFFFFA463 == 1 (closed-loop enabled)
  → Check 0xFFFFB33F or 0xB340 (O2 sensor online)
  → Adjust 0x6ACE4 (O2 gain) if convergence is too fast/slow
  → Target: ±0.5 AFR oscillation within 5 seconds

STEP 3: Cold-start smoothness (30-70°C)
  → Verify 0xFFFFA6B8 cold-start flag cycles properly
  → Log 0xFFFFB494 (corrected MAF) vs. load
  → Adjust 0x69FAE-0x69FEA if stumble/surge occurs
  → Verify no stall < 1000 RPM

STEP 4: Transient enrichment (WOT ramp)
  → Monitor 0xFFFFB4A4 (accel flag)
  → Verify 0xFFFFB4A0 (transient correction) > 0 during pedal
  → Adjust 0x6AD0C if bog/hesitation detected

STEP 5: Emissions validation (dyno 4-gas)
  → CO should be 0.2-0.5%  → Adjust 0xFFFFB5B8 richer if too high
  → HC should be < 100 ppm → Adjust cold-start richness
  → NOx should be < 800 ppm → Adjust transient lean if possible

================================================================================
COMMON ISSUES & DIAGNOSTICS
================================================================================

SYMPTOM: Engine running rich (black soot, high CO)
  CAUSE: 0xFFFFB5B8 too high or 0x69FAE stuck enrichment
  CHECK: @(0xFFFFB494) reading (should be < 0x41A00000)
  FIX:   Decrease 0xFFFFB5B8 by 0.5 AFR, or reduce 0x69FAE

SYMPTOM: Engine running lean (detonation, white spark plugs)
  CAUSE: 0xFFFFB5B8 too low or 0x6ACE4 overcorrecting
  CHECK: @(0xFFFFA498) saturation accumulator (should be < 0x80)
  FIX:   Increase 0xFFFFB5B8 by 0.5 AFR, reduce 0x6ACE4 gain

SYMPTOM: Idle surge/surging (800-1200 RPM)
  CAUSE: Transient enrichment oscillating via 0x6AD0C
  CHECK: @(0xFFFFB4A4) fluttering (should be 0 at idle)
  FIX:   Smooth 0x6AD0C table or increase deadband in sensor filter

SYMPTOM: Cold start stall (< 20°C)
  CAUSE: 0x69FAE-0x69FEA enrichment insufficient
  CHECK: @(0xFFFFA6B8) = 1 for 5+ seconds after cranking
  FIX:   Increase cold-start tables by 0.2-0.3 AFR richness

SYMPTOM: Check Engine Light - P0100 (MAF sensor malfunction)
  CAUSE: 0xFFFFB494 out of expected range or changing too fast
  CHECK: dtc_p0100_maf_46DA0 function sets DTC if plausibility fails
  FIX:   Verify MAF sensor wiring, check linearization table 0x69F72

SYMPTOM: Asymmetric rotor wear/fouling
  CAUSE: 0x6996E (rotor2 stoich offset) mistuned
  CHECK: Plug #1 vs #2 fouling pattern
  FIX:   Adjust 0x6996E to richen rotor2 if fouled, or lean if glazed
