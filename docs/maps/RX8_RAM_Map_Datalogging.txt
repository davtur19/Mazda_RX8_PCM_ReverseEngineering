================================================================================
MAZDA RX-8 S1 PCM — RAM VARIABLE MAP FOR DATALOGGING
ROM: N3J1EL (60E1D400) — EU 6-port 231hp MT, 2004-2008
RAM: 0xFFFF6000 - 0xFFFFDFFF (32KB)
Access: OBD SID 0x23 (ReadMemoryByAddress) after session 0x85 + security unlock
         ⚠ UNVERIFIABLE: Firmware-derived session codes (0x81/0x85/0x87) — no public source confirms these values
================================================================================

Legend:
  [CONFIRMED]   = Verified via IDA disassembly of specific function
  [INFERRED]    = Identified from function context, high confidence
  [SPECULATIVE] = Based on address pattern/proximity, needs verification
  Size: B=byte(1), W=word(2), F=float(4)

CRITICAL VERIFICATION REQUIREMENT:
  ⚠ ALL RAM ADDRESSES ARE DERIVED FROM STATIC ROM ANALYSIS
    All RAM addresses in this document are determined from disassembly of the ROM image.
    SH7055 uses 16-bit displacement addressing, which makes direct verification from
    disassembly incomplete. RAM addresses CANNOT be fully verified without runtime data.
    REQUIRED VERIFICATION METHODS:
      1. SID 0x23 dump (ReadMemoryByAddress after session 0x85 + security unlock)
      2. CAN bus correlation with live telemetry
      3. Hardware breakpoint analysis on actual ECU hardware
    Use this map for research guidance only. Validate all critical addresses before
    applying to real vehicles or production systems.

KNOCK DETECTION (verified from knock_detection_processing 0xF770)
================================================================================
  Architecture: Per-rotor distributed structures, 0x58 bytes each.
  Base: 0xFFFFA4B4. Addressed via r13 + relative offset @(N,r13).
  Note: 4 knock channels map to: Leading_Rotor1, Trailing_Rotor1, Leading_Rotor2, Trailing_Rotor2

  Per-Rotor Structure Layout (0x58 bytes each):
    Rotor Housing Channel 0 (Leading_Rotor1): 0xFFFFA4B4 - 0xFFFFA50B  (retard output at 0xFFFFA4B8)
    Rotor Housing Channel 1 (Trailing_Rotor1): 0xFFFFA50C - 0xFFFFA563  (retard output at 0xFFFFA510)
    Rotor Housing Channel 2 (Leading_Rotor2): 0xFFFFA564 - 0xFFFFA5BB  (retard output at 0xFFFFA568)
    Rotor Housing Channel 3 (Trailing_Rotor2): 0xFFFFA5BC - 0xFFFFA613  (retard output at 0xFFFFA5C0)

  0xFFFFA4B4  D  [CONFIRMED]  Knock Channel 0 Base / State
              Base address for per-rotor knock state machine.
              Each 0x58-byte structure contains: state word, retard level,
              recovery counter, knock event history, threshold comparisons.

  0xFFFFA4B8  F  [CONFIRMED]  Knock Retard Output — Rotor Housing Channel 0
              Per-rotor ignition retard from knock detection.

  0xFFFFA510  F  [CONFIRMED]  Knock Retard Output — Rotor Housing Channel 1
              Per-rotor ignition retard from knock detection.

  0xFFFFA564  B  [CONFIRMED]  Knock Counter / Enable — Rotor Housing Channel 2
              Knock event counter for rotor housing channel 2.

  0xFFFFA568  F  [CONFIRMED]  Knock Retard Output — Rotor Housing Channel 2
              Per-rotor ignition retard from knock detection.

  0xFFFFA56D  B  [CONFIRMED]  Knock Retard Level Byte
              Part of per-rotor knock state structure.
              Controls magnitude of knock-triggered ignition retard.

  0xFFFFA570  D  [CONFIRMED]  Knock State Register
              Written by knock_detection_processing (0xF770).
              4-byte state word tracking knock event history and retard level.
              Threshold constant: 0x1680000 loaded for comparison
              (this is a register constant value, not a ROM address).

  0xFFFFA571  B  [CONFIRMED]  Knock Detection Enable Flag
              Per-rotor enable/disable for knock detection processing.

  0xFFFFA56E  B  [CONFIRMED]  VVT Solenoid Status (knock context)
              Read by knock detection to adjust thresholds during VVT operation.

  0xFFFFA5C0  F  [CONFIRMED]  Knock Retard Output — Rotor Housing Channel 3
              Per-rotor ignition retard from knock detection.

  NOTE: 0xFFFFAA7A/AA78/AA76 listed in Spark Advance section are used by
  spark_advance_calc_main for knock COMPENSATION in timing, not by the
  knock_detection_processing function itself.
