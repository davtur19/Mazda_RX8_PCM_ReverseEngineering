CRITICAL ACCESS PATTERNS & TIMING
================================================================================

ATOMIC OPERATIONS (MUST NOT BE INTERRUPTED):
    - Task state flags (0xFFFFA1C0–0xFFFFA1C9): Read-Modify-Write cycles
    - Rotor timing state (0xFFFFA1D8–0xFFFFA208): Accessed by interrupt + main
    - Lambda PID state (0xFFFFA328–0xFFFFA38C): Requires consistent snapshot
    - Knock detection (0xFFFFA4B4–0xFFFFA571): Per-rotor counter updates

UPDATE FREQUENCIES:
    - Sensor state: Every ADC cycle (≈10ms)
    - Fuel calculation: Every 25-50ms
    - Ignition timing: Every 10-50ms
    - Lambda PID: Every 100-200ms
    - CAN messages: Every 100-300ms (300ms periodic cycle)
    - Knock detection: Per-rotor at eccentric shaft angle (≈2.5ms at 6000 RPM)

INITIALIZATION:
    - Cold boot: All RAM cleared by cold_start_ram_init (0x5B0)
    - Warm reset: RAM preserved, status checked at 0xFFFFDFFC
    - Startup sequence: pcm_init_sequence (0xDD30) calls 9 init routines

WATCHDOG & SAFETY:
    - Watchdog counter at 0xFFFFDFF4 (counts down, reset by watchdog_service)
    - Checksum validation at 0xFFFFA3F9 (detects RAM corruption)
    - Stack guard at 0xFFFFDFA0 (watchdog if SP underflows)
    - Warm reset magic at 0xFFFFDFFC distinguishes power-on vs restart
